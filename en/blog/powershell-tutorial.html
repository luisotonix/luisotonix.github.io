PowerShell for Administrators: From Basics to Advanced

üë§ Lu√≠s Otoni üìÖ October 28, 2025 ‚è±Ô∏è 20 min read üìÅ Automation

PowerShell is the most powerful tool in a Microsoft administrator's arsenal. Whether managing Active Directory, Microsoft 365, Azure or Windows Server, mastering PowerShell is the difference between executing tasks manually and automating your infrastructure.

1. PowerShell Fundamentals

Why PowerShell?

  • Automation: Execute repetitive tasks automatically
  • Consistency: Scripts produce predictable results
  • Scale: Manage thousands of objects with one command
  • Integration: Works with Azure, M365, AD, Exchange, and more

Cmdlets: The Foundation

Cmdlets follow the Verb-Noun pattern:

Get-Process      # Gets processes
Set-Location     # Sets location
New-Item         # Creates new item
Remove-Item      # Removes item

2. Working with Objects

PowerShell works with objects, not text. This changes everything.

Pipeline: The Power of |

# List stopped services
Get-Service | Where-Object {$_.Status -eq 'Stopped'}

# Get top 5 memory-consuming processes
Get-Process | Sort-Object WorkingSet64 -Descending | Select-Object -First 5

4. Scripts and Functions

function Get-DiskSpace {
    [CmdletBinding()]
    param(
        [string[]]$ComputerName = $env:COMPUTERNAME
    )
    
    foreach ($computer in $ComputerName) {
        Get-WmiObject Win32_LogicalDisk -ComputerName $computer -Filter "DriveType=3" |
        Select-Object @{N='Computer';E={$computer}},
                      DeviceID,
                      @{N='SizeGB';E={[math]::Round($_.Size/1GB,2)}},
                      @{N='FreeGB';E={[math]::Round($_.FreeSpace/1GB,2)}}
    }
}

5. PowerShell for Microsoft 365

# Install required modules
Install-Module Microsoft.Graph -Scope CurrentUser

# Connect to Microsoft Graph
Connect-MgGraph -Scopes "User.Read.All", "Group.ReadWrite.All"

# List all users
Get-MgUser -All | Select-Object DisplayName, UserPrincipalName, AccountEnabled

6. PowerShell for Azure

# Connect
Connect-AzAccount

# List VMs
Get-AzVM | Select-Object Name, ResourceGroupName, Location

# Start VM
Start-AzVM -ResourceGroupName "RG-Production" -Name "VM-Web01"

7. Active Directory with PowerShell

# Users who haven't logged in for 90 days
$90DaysAgo = (Get-Date).AddDays(-90)
Get-ADUser -Filter {LastLogonDate -lt $90DaysAgo} -Properties LastLogonDate |
Select-Object Name, SamAccountName, LastLogonDate

Conclusion

PowerShell is an essential skill for any Microsoft system administrator. Start automating simple tasks and gradually evolve to more complex scripts.
