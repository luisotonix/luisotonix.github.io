The Challenge

The company faced multiple security challenges after a rapid migration to Azure. The environment lacked centralized visibility over threats, consistent security policies, and compliance with financial sector regulations (GDPR, PCI-DSS).

The main problems identified included: resources exposed to the internet without adequate protection, absence of real-time security monitoring, excessive permissions on service accounts, and lack of governance over resource creation.

The Solution

I implemented a layered security strategy, starting with enabling Microsoft Defender for Cloud (formerly Azure Security Center) with Standard tier on all resources.

Phase 1: Assessment and Quick Wins

  • Complete Secure Score analysis and recommendation prioritization
  • Critical vulnerability remediation (exposed ports, missing MFA)
  • Azure Policy implementation to enforce basic standards
  • Alert configuration for suspicious activities

Phase 2: Azure Sentinel (SIEM/SOAR)

  • Centralized Log Analytics workspace deployment
  • Data connector integration (Azure AD, O365, Azure Activity)
  • Custom analytics rules for the financial sector
  • Automated playbooks for incident response

Phase 3: Governance Framework

  • Management Groups structure for environment segregation
  • Azure Blueprints for standardized deployments
  • GDPR and PCI-DSS compliance policies
  • Refined RBAC with least privilege principle

Phase 4: Advanced Protection

  • Microsoft Defender for Servers on all VMs
  • Defender for SQL for databases
  • Defender for Storage for data protection
  • Just-in-Time VM Access for administration
